# A/B Test: Cookie Cats

Пет-проект по анализу A/B-теста на датасете [Mobile Games A/B Testing - Cookie Cats](https://www.kaggle.com/datasets/mursideyarkin/mobile-games-ab-testing-cookie-cats).

## Контекст

Cookie Cats — мобильная игра-головоломка. В игре есть "ворота" (gate), которые заставляют игрока подождать или заплатить, чтобы продолжить прохождение. Изначально ворота стояли на 30 уровне. Разработчики хотят понять: если сдвинуть ворота на 40 уровень, изменится ли удержание игроков?

- `gate_30` — контрольная группа (ворота на 30 уровне)
- `gate_40` — тестовая группа (ворота на 40 уровне)
- `retention_1` / `retention_7` — вернулся ли игрок в игру через 1 / 7 дней

## Методология

1. Проверка дизайна эксперимента: размеры групп, отсутствие дублей пользователей.
2. Сравнение retention rate между группами.
3. Формулировка гипотез:
   - H0: доля вернувшихся игроков одинакова в `gate_30` и `gate_40`
   - H1: доли различаются
4. Двухвыборочный z-тест для пропорций (`statsmodels.stats.proportion.proportions_ztest`), доверительные интервалы (`proportion_confint`), уровень значимости `alpha = 0.05`.
5. Bootstrap-проверка результата как альтернатива параметрическому тесту.

## Результаты

> Ожидаемый результат по открытым разборам этого датасета — проверьте на фактических цифрах после запуска ноутбука.

| Метрика | gate_30 | gate_40 | p-value | Значимо? |
|---|---|---|---|---|
| retention_1 | ~44.8% | ~44.2% | ~0.07 | Нет |
| retention_7 | ~19.0% | ~18.2% | < 0.05 | Да |

Bootstrap подтверждает: `gate_30` лучше по `retention_7` в ~96% пересэмплированных выборок.

**Рекомендация:** переносить ворота с 30 на 40 уровень не стоит — значимой разницы в 1-дневном удержании нет, а по 7-дневному удержанию `gate_30` статистически значимо лучше.
